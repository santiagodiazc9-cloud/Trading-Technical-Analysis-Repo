### CRITICAL
1. **`requirements-bootstrap.txt:5`** — `setuptools>=65` permits installation of versions vulnerable to **CVE-2025-47273** (GHSA-5rjg-fvgr-3xxf): path traversal in `PackageIndex.download` → arbitrary file write / RCE during `pip install`. Fixed in setuptools 78.1.1. **Suggested fix**: `setuptools>=78.1.1`.

### HIGH
1. **`requirements-api.txt:1`** — `fastapi>=0.110` pulls Starlette `<1.0.1` affected by **CVE-2026-48710 "BADHOST"** (GHSA-86qp-5c8j-p5mr): host-header auth bypass. **Suggested fix**: `fastapi>=0.118`.
2. **`api/main.py:45-50`** — `CORSMiddleware(allow_origins=["*"], allow_methods=["*"], allow_headers=["*"])` combined with no visible auth on POST endpoints (`/approve/{id}`, `/pause`, `/session`). Any browser visiting a malicious page can approve setups → trade execution. `allow_credentials=False` (default) limits damage today; if anyone flips that to True, this becomes CRITICAL. **Suggested fix**: restrict origins + add API-key check on POST endpoints.
3. **`scripts/discord_bot_cloud.py:70`** — **REGRESSION** from 2026-05-16 scan (ClickUp `869daw4dd`, still open). GH token embedded in git remote URL (`https://{token}@github.com/...`) and persisted to `.git/config`. **Suggested fix**: use `GIT_ASKPASS` or credential helper.

### MEDIUM
1. **`scripts/daytrader/ml_model.py:127`** — `pickle.load(f)` deserializes model files from `MODELS_DIR/<symbol>.pkl`. If attacker controls `MODELS_DIR` write OR `symbol` path traversal → RCE on next load. Models dir is `.gitignore`d but lives on cloud worker disk. **Suggested fix**: switch to `joblib` with HMAC integrity, sanitize symbol against `^[A-Z]{1,5}$` before path build.
2. **urllib3 transitive** — CVE-2025-66418 decompression-chain DoS, affects `<2.5.1`. Pulled via `httpx`/`requests` chain. **Suggested fix**: pin `urllib3>=2.5.1` in `requirements.txt`.
3. **`requirements.txt:3`** — `numpy>=1.24.0` permits EOL 1.x series (EOL Sept 2025, no more security patches). **Suggested fix**: `numpy>=2.0` after verifying `pandas`/`ta` compatibility.

### LOW
1. **`scripts/discord_bot_cloud.py:60-74`** — three `subprocess.run(..., check=True)` calls missing `timeout=`. A stalled git/SSH op would hang the cloud worker indefinitely. **Suggested fix**: add `timeout=30` to each.

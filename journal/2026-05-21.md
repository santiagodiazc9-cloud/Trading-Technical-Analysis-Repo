## Lessons
- A trade can be committed to one memory file (`open_positions.md`) and silently skipped in another (`trade_log.json`) when a routine completes partially. EOD review must reconcile the two — `trade_log.json` is the cumulative ledger, and a missing entry corrupts every downstream stat (weekly count, win rate, confidence calibration). Tonight's reconciliation caught a 1-day-old gap; the durable fix is for the execution routine to write both files atomically.
